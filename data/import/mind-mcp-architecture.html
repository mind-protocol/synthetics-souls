1. Vue d'ensemble

Le système est une boucle fermée auto-calibrante :

  • Les inputs sensoriels et substances alimentent un modèle cognitif
  • Le modèle prédit le comportement
  • Le comportement observé est comparé à la prédiction
  • L'erreur calibre le modèle

2. Layers d'Abstraction

Layer 0 — Inputs (Acquisition)

Fonction: Capturer les données brutes du monde et de l'état interne.

Layer 1 — Features Brutes (Extraction)

Fonction: Extraire les caractéristiques de bas niveau.

Luminosité, couleurs, edges, volumes, accélérations, courbes pharmacocinétiques...

Layer 2 — Patterns (Agrégation)

Fonction: Regrouper les features en patterns reconnaissables.

Formes géométriques, segments parole, gestes, postures, phases de courbes substances...

Layer 3 — Objets (Reconnaissance)

Fonction: Identifier des entités discrètes.

Personnes, objets, lieux, actions en cours, états émotionnels visibles...

Layer 4 — Concepts (Sémantique)

Fonction: Mapper sur des catégories significatives.

flowchart LR subgraph SURVIVAL["Catégories Survie"] SV1["🦁 Prédateur"] SV2["🍎 Nourriture"] SV3["🏠 Abri"] SV4["💕 Partenaire"] end subgraph COGNITIVE["Catégories Cognitives"] CG1["📋 Tâche"] CG2["🎯 Objectif"] CG3["⚠️ Problème"] end

Layer 5 — Noyau Cognitif (Intégration)

Fonction: Combiner toutes les informations pour produire l'état cognitif.

QI_effective[i] = f(QI_base[i], QI_perception[i])
QI_final[i] = QI_effective[i] × global_reduction × sphere_modifiers[i]
cognitive_state = {QI_final[], spheres[], survival_balance, concepts[]}

Layer 6 — Prédiction (Décision)

Fonction: Prédire le comportement le plus probable.

P(action | état) = base_prob × survival_mod × habit_mod × opportunity_mod
predicted_action = argmax(P)

Layer 7 — Output (Action)

Fonction: Exécuter l'action et la rendre observable.

Décision → Planning moteur → Mouvement → Comportement observable

Layer 8 — Calibration (Apprentissage)

Fonction: Comparer prédiction et observation, ajuster le modèle.

4. Composants Détaillés

4.1 Pharmacokinetic Engine

Responsabilité: Calculer les niveaux de substances en temps réel.

Modèle two-compartment:

dA/dt = -k_abs × A     (absorption)
dC/dt = k_abs × A - k_elim × C     (compartiment central)

level(t) = dose × bioavailability × (e-k_elim×t - e-k_abs×t) / (k_abs - k_elim)

4.2 QI Processor

Responsabilité: Calculer les QI finals à partir des bases et de l'état actuel.

flowchart LR B["QI Base\n(test)"] --> E["QI Effectif"] P["QI Perception\n(self)"] --> E E --> F["QI Final"] R["Global\nReduction"] --> F S["Sphere\nModifiers"] --> F

4.3 Behavior Predictor

Responsabilité: Prédire l'action la plus probable.

FOR each action in action_space:
    P(action) = base_probability(action, cognitive_state)
    P(action) *= survival_modifier(action, survival_balance)
    P(action) *= habit_modifier(action, action_history)
    P(action) *= opportunity_modifier(action, active_concepts)
normalize(P)
predicted_action = argmax(P)

4.4 Calibration Engine

Responsabilité: Apprendre des erreurs de prédiction.

IF prediction ≠ observation:
    error = compute_error(prediction, observation)
    candidates = identify_candidate_variables(error)
    
    FOR each var in candidates:
        adjustment = error × (1 - var.certainty) × explanatory_power
        var.value += clip(adjustment, -MAX_ADJ, +MAX_ADJ)
        var.certainty = update_certainty(observation_count)
    
    log_calibration(error, adjustments)
